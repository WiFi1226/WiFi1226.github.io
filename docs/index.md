# 关于 Econometric Technology Library

**Econometric Technology Library** 是一份专注于计量经济学技术方法、实证工具与编程实践的个人技术博客。

博客名中的“Library”承载着两层含义：其一，它是一份系统性梳理计量方法论的“知识索引”，将分散在教材、论文和代码仓库中的核心思想加以提炼与整理；其二，也暗合计量经济学的传统表达——例如 Greene 教授的经典教材 **Econometric Analysis** 与 Davidson / MacKinnon 的 **Econometric Theory and Methods** 中时常出现的“library”之说，以博采众长的姿态，将理论与应用融为一炉。这里没有艰深的理论推演强塞给读者，而是力图从实际研究者的视角出发，把那些写作时绕不开、教材里却讲不透的技术细节讲清楚。

## 🔎 博客涵盖哪些方面？

- **因果推断方法**：DID 及其现代扩展（交错 DID、组别—时间 ATTs 等）、工具变量、断点回归、合成控制法等核心识别策略；
- **时间序列分析**：ARIMA / VAR、协整与 ECM、GARCH 类波动率模型、状态空间建模、贝叶斯 VAR 等；
- **面板数据模型**：固定 / 随机效应、异质性建模、动态面板、长面板与非线性面板；
- **编程与软件实现**：以 **Stata**、**R**、**Python** 为主要工具，辅以少量 **MATLAB** 和 **Julia**，提供可复现的代码示例与实证中的技术陷阱提示；
- **机器学习与计量交叉**：因果森林、双重机器学习（DML）、LASSO 用于变量选择、预测与高维控制等；
- **论文复现与数据实践**：精选经典或前沿实证研究的复现代码，帮助读者将方法落地到自己的数据中；
- **实证研究的工作流**：数据清洗、可视化、结果汇报与自动化文档的工程化技巧。

## 💡 技术特色

博客强调从应用研究的真实场景出发：

- **可复现性优先**：每篇涉及代码的博文均尽可能提供可直接运行的示例，并注明依赖环境；
- **对标前沿，不盲从**：在介绍新方法时，会同时提示模型的适用条件与潜在局限，帮助读者避免方法滥用；
- **工程与直觉兼顾**：既给出软件层面的实现细节，也解释方法背后的经济直觉——不必迷恋繁琐的数学推导，但要以精准的直觉驾驭模型。

## ✍️ 写作风格与理念

博客在写作上力求 **简洁、直白、贴近实战**。每一篇文章都试图回答一个具体的技术问题：这个问题在实证研究中为什么会出现？有哪些解决思路？它们的差异和适用场景是什么？这样的风格深受 **Angrist & Pischke 的“极简计量”精神**以及 **Wooldridge 在各类讲义与讨论中那种清晰直白的讲解方式**的影响——不堆砌术语，而是真正帮助读者理解“方法在做什么”。

如果你也曾因计量软件中的一行报错而反复排查，为因果识别中的平行趋势检验而苦恼，或在写论文时面对方法部分的“规范表述”无从下手——那么这个博客或许正是你需要的。

## 📌 友情提示

博客中的所有内容均为作者学习与实证工作中的经验总结，作者并非计量理论专家，而是和你一样需要每天与数据和模型打交道的研究者或数据分析从业者。方法的选择与结果的解读请结合自身研究问题谨慎判断，欢迎交流、讨论与指正。

> 经济计量不能回答所有问题，但好的经济计量至少可以让人们看清“相关”与“因果”之间那道并不浅的沟壑。
